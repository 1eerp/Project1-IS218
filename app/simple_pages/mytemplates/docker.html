What is Docker?

Docker is platform that allows us to build, run and ship running applications.

What is an Image?

The simplest way to put it would be, an image is a blueprint to build a container, and the image would conatin what libraries, tools or other images are needed to build that container.

What is a container?

A container in the simplest terms would be an OS-level virtualization, and they are many of times better than kernel level virtual machines, since it is much more easier to replace a container than a computer running virtual machines.

Docker commands and examples
docker run -dp 80:80 ubuntu:latest

Find the image called ubuntu:latest and use it to build and run a container, and port 80 of your physical machine is being used by the the container's port 80.

docker build -t "tagname"

This command builds an image by using the instructions in a file called Docker

Dockerfile & docker-compose.yml

A Dockerfile has instructions that automates the proecess of creating an image, so that image can be used to create a container that runs an application, and a docker-compose.yml is the file that automamtes the process of building and running a container ("docker compose" is the command too use to use the docker-compose.yml file to build and run the container).

docker ps

Shows what containers are currently running (use -a option to show all containers)

docker stop

Use "docker stop 'container id/name'" to stop the container if it is running at that moment.

docker rm

Use "docker rm 'container id/name' " to delete a container.

docker image rm

Use this command to delete any docker image that's on your system.

docker logs

Use "docker logs 'container id/name'" to retrieve the logs that the container has made so far (use the -f option after "docker logs" to follow the logs as they are happening).

Run project locally...

If you have a docker-compose.yml file you can run the docker compose up --build command to run and build the container which will run your app, and to if the docker-compose.yaml file runs the container with the "-i" option the you will be able to use the containers terminal from the terminal you ran the compose command from. If you do not have a compose file you can use the "docker run" command with the "-i" option to interact with the containers terminal on your terminal.

Dockerhub Repositories

Dockerhub is a library for container images, you can pull images from DockerHub to your local machine by using the command "docker pull". If you would like to upload your images to DockerHub, login in to docker via the terminal or the desktop app and then create a DockerHub respository and replace the name of the image you want to push with your DockerHub username and repository by using "docker tag", then use the command "docker push 'image name'".
